## 1.1 Назначение
Этот документ является основным источником с требованиями к проекту по продаже автомобилей.

<a name="business_requirements"/>

## 1.2 Бизнес-требования

<a name="initial_data"/>

### 1.2.1 Исходные данные
В наше время все пользуются автомобилями. Они стали важной частью жизни любого человека. И сильнее всего запоминается момент покупки машины. Но для покупки автомобиля в реальности требуется выполнять много лишних действий. Требуется ездить в различные автосалоны, выбирать машину только из определенного каталога и сравнивать большое количество цен, чтобы сделать самый оптимальный и целесообразный выбор. Для этого требуется приложение, в котором можно будет всего в пару кликов выполнить большинство этих сложных действий.

<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Как базовые пользователи, так и компании реселлеры выполняю слишком большое количество ненужно работы по выбору нужного авто. Для этого приложение дожно быть простым и максимально автоматизированным по подбору автомабилей, но при этом имеющим возможность выпонять базовые действия для пользователя.

<a name="project_boundary"/>

### 1.2.3 Границы проекта
Приложение по подбору автомобилей должно реализовать backend интернет магазина по покупке автомобилей, и при этом автоматизировать покупку автомобилей для реселлеров.

# 2 Требования пользователя

<a name="software_interfaces"/>

## 2.1 Программные интерфейсы
Приложение должно работать на основе docker-контейнера. В приложении должен использоваться Django версии 4.1.1. Должно быть написано на языка Python 3.10 и данных храниться в базе данных PostgreSQL версии 14.4. Должна быть реализовать  JWT-регистрация пользователей. Работа автосервисов и диллеров должныть быть автоматизирована.

<a name="user_interface"/>

## 2.2 Интерфейс пользователя
При переходе на адрес автосервиса или диллера должен отражаться системное окно для редактированя, просмотра и удаления записей в базе данных.  
![Интерфейс редактирования автосервисов/](../../Images/Mockups/StartPage.png)  

При переходе на андрес регистрации, должно отображаться окно регистрации
![Диалог первичной настройки](../../Images/Mockups/Configuring.png)  

Так должен выглядеть окно, при переходе к конктеному объекту базы данных

# 3 Системные требования

<a name="functional_requirements"/>

## 3.1 Функциональные требования

<a name="main_functions"/>

### 3.1.1 Основные функции

<a name="user_buy_car"/>

#### 3.1.1.1 Покупка автомобиля
**Описание.** Пользователь имеет возможность покупки автомобиля.

| Функция | Требования | 
|:---|:---|
| Покупка автомобиля | Приложение должно запросить объект автомобиля в базе данных и порвести процесс покупки |

<a name="jwt_auth"/>

#### 3.1.1.2 Аутентификация по JWT-токену
**Описание.** Пользователь имеет возможность аутентифицироваться по токену
 
| Функция | Требования | 
|:---|:---|
| Аутентификация по JWT-токену | Приложение должно получить accese token и refresh токен из дазы данных, чтобы корректно аутентифицировать пользователя.|

<a name="edit_data"/>

#### 3.1.1.3 Редактирование объектов базы данных администратором
**Описание.** Пользователь имеет возможность редактирования напоминания
 
| Функция | Требования | 
|:---|:---|
| Редактирование объектов | Приложение должно иметь возможность редактирования и удаления существующих обектов от имени администратора.|

<a name="list_of_objects"/>

#### 3.1.1.4 Просмотр машин, автосервисов, диллеров
**Описание.**  Пользователь имеет возможность просмотреть все существующие объекты дазы данных(автосервисы, автомобили, диллеров).

| Функция | Требования | 
|:---|:---|
| Просмотр объектор | Приложение должно вывести информацию о всех объектах текущей категории. |

<a name="sale_history"/>

#### 3.1.1.5 История продаж
**Описание.** Автосервисы автоматически имеют возможность обслеживать нужные машины у диллеров и покупать их автоматически.

| Функция | Требования | 
|:---|:---|
| История продаж | Приложение должно продемонстрировать всю информацию о проданных автомобилях(дату, автомобить, покупалеля, цену). |

<a name="auto_buy"/>

#### 3.1.1.6 Автопокупка автомобилей
**Описание.** Покупка автомобилей у диллеров автосервисами происходит автоматически.

| Функция | Требования | 
|:---|:---|
| Автопокупка автомобилей | Приложение должно автоматические в определенные интервалы времени найти подходящие для них автомобили у диллеров и купить их. |

## 3.2 Нефункциональные требования

<a name="quality_attributes"/>

### 3.2.1 Атрибуты качества

<a name="requirements_for_ease_of_use"/>

#### 3.2.1.1 Требования к удобству использования
1. Способ способ работы над обектами от имени администратора должен быть реализован с помощью DRF и иметь обширный функционал;
2. Обратная связь с пользователем

<a name="security_requirements"/>

#### 3.2.1.2 Требования к безопасности
1. Приложение предоставляет возможность просмотра и редактирования только супер пользователю и владельцу записи.
2. Приложение перед предъявления пользователю прав на редактирование должно удостовериться, что пользователь, тот за кого себя выдает.
